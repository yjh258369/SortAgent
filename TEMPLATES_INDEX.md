## 🚀 立即开始

### 最简单的使用方式：

1. **打开** [QUICK_USE.md](./QUICK_USE.md)
2. **复制** 第一段"一键执行"提示词
3. **替换** `[目标目录]` 为你的文档路径
4. **粘贴** 到 Claude Code 或任何 AI 编程助手
5. **等待** 执行完成

### 示例：

```bash
# 原始提示词
请帮我整理 [目标目录] 的所有 .md 文件...

# 替换后
请帮我整理 ./my-project-docs/ 的所有 .md 文件...
```

## 📝 自定义建议

### 1. 调整摘要长度
```
# 默认 200 字
summary_length: 200

# 改为 150 字（更简洁）
"为每个文件生成 150 字摘要"

# 改为 300 字（更详细）
"为每个文件生成 300 字摘要"
```

### 2. 修改重要性阈值
```
# 默认删除 <2 星
importance_threshold: 2

# 更严格（删除 <3 星）
"删除重要性 <3 星的文件"

# 更宽松（只删除 1 星）
"删除重要性 1 星的文件"
```

### 3. 添加自定义标签
```
# 在提示词中添加
"额外打上以下标签：
- 项目阶段：规划/开发/测试/上线
- 受众：开发/运维/产品/全部
- 状态：草稿/审核/已发布"
```

## ✨ 最佳实践

1. **首次使用**：先用小目录测试（5-10 个文件）
2. **定期维护**：每周或每月执行一次
3. **版本控制**：使用 Git 管理文档变更
4. **备份重要**：始终启用备份功能
5. **标签一致**：建立标准化的标签体系
