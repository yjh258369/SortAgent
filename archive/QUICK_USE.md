## 🚀 一键执行（推荐）

```
请帮我整理 [目标目录] 的所有 .md 文件：

1. 扫描并列出所有文件
2. 为每个文件生成 200 字摘要
3. 打标签：时间、类型、重要性（1-5星）
4. 识别重复内容（相似度 >50%）
5. 创建 README.md 索引（按类型和重要性分组）
6. 创建 FILE_ANALYSIS.md 分析报告
7. 重命名文件为 YYYY-MM-DD_description.md 格式
8. 删除重要性 <2 星的文件（需确认）
9. 创建备份到 archive/ 目录

完成后生成执行日志。
```

---

## 📋 分步执行

### 步骤 1：生成摘要
```
读取 [目录] 下所有 .md 文件，为每个文件生成 200 字摘要。
摘要要求：提取核心信息、突出技术细节、使用简洁语言。
```

### 步骤 2：打标签
```
为每个文件打标签：
- 时间：创建日期
- 类型：配置文档/技术日志/会话记录/API文档
- 重要性：⭐(1星) 到 ⭐⭐⭐⭐⭐(5星)
- 主题：根据内容提取关键词
```

### 步骤 3：识别重复
```
对比所有文件摘要，找出相似度 >50% 的内容。
输出：重复文件对、重复度、重复内容、处理建议。
```

### 步骤 4：生成索引
```
创建 README.md，包含：
- 📂 文档列表（按类型分组）
- ⭐ 重要性分组（5星到1星）
- 🏷️ 标签索引
- 🔍 快速查找指南
- ⚠️ 待办事项
```

### 步骤 5：优化结构
```
优化文件结构：
1. 创建备份：cp *.md archive/
2. 重命名：YYYY-MM-DD_description.md
3. 删除低价值文件（<2星）
4. 生成操作日志
```

---

## ⚙️ 自定义参数

修改这些参数适应你的需求：

```
目标目录：./docs/          → 你的文档目录
文件类型：.md              → .md, .txt, .json
摘要长度：200 字           → 100-500 字
重要性阈值：2 星           → 1-3 星
重复阈值：50%              → 30-80%
需要确认：是               → 是/否
```

## 🚨 快速问题解决

**文件太多？** → 分批处理，每次 20-30 个文件

**误删文件？** → 从 archive/ 恢复

**标签不准？** → 在提示词中添加自定义标签规则

**链接失效？** → 使用相对路径，检查文件名
